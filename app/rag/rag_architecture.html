📁 services

RAG 실행 서비스 계층

개별 기술 단위의 로직을 수행하며, 주로 RagStateBuilder에서 호출됩니다. 문서 인덱싱, 벡터 임베딩, 지식 검색, 답변 생성의 4가지 서비스로 구성됩니다.

📄 indexing_service.py 문서 인덱싱
원본 문서를 청킹(Chunking)하고 메타데이터를 추출하여 저장 준비합니다.
함수설명InputOutput
process_and_index(raw_data) 원본 문서를 청킹하고 메타데이터를 추출하여 저장 준비 PDF, 텍스트 파일 등의 원본 데이터 인덱싱 완료 상태 및 문서 ID
update_index(document_id, new_data) 기존 인덱싱된 데이터를 수정 문서 ID, 수정 데이터 업데이트된 인덱스 상태
📄 embedding_service.py 벡터 변환
텍스트를 고차원 벡터로 변환하고 벡터 간 유사도를 계산합니다.
함수설명InputOutput
get_embeddings(text_list) 텍스트를 고차원 벡터로 변환 텍스트 리스트 또는 쿼리 벡터 데이터 (List[float])
calculate_similarity(vec1, vec2) 벡터 간의 유사도 계산 벡터 두 개 유사도 점수 (float)
📄 retrieval_service.py 지식 검색
쿼리와 가장 유사한 문서 조각을 검색하고, 검색 결과의 순위를 재정렬하여 정확도를 향상합니다.
함수설명InputOutput
retrieve_relevant_docs(query, k) 쿼리와 가장 유사한 k개의 문서 조각을 검색 사용자 질문(Query), 검색 개수(k) 검색된 관련 문서 리스트
rerank_documents(query, docs) 검색된 결과의 순위를 재정렬하여 정확도 향상 질문, 문서 리스트 재정렬된 문서 리스트
📄 rag_generation_service.py 답변 생성
검색된 지식과 질문을 결합하여 최적의 프롬프트를 구성하고, LLM을 호출해 최종 답변을 생성합니다.
함수설명InputOutput
construct_prompt(context, question) 검색된 지식과 질문을 결합하여 최적의 프롬프트 구성 검색 결과(Context), 질문 최적화된 프롬프트 문자열
invoke_llm(prompt) LLM을 호출하여 최종 답변 생성 프롬프트 문자열 생성된 최종 텍스트 답변
📁 adapters

RAG 외부 인터페이스 계층

외부 라이브러리(LangChain, LlamaIndex 등)나 API와의 의존성을 분리하는 완충 지대입니다. 추상 인터페이스를 통해 실제 구현체와 테스트용 Mock을 교체 가능하게 합니다.

📄 rag_adapter.py 추상 인터페이스
모든 RAG 어댑터가 따라야 할 추상 인터페이스를 정의합니다.
함수설명InputOutput
run_workflow(input_data) RAG 워크플로우를 실행하는 추상 메서드 사용자 입력 객체 최종 응답 객체
📄 real_rag_adapter.py 실제 구현체
실제 LangGraph·외부 엔진에 연결하고, OpenAI / Anthropic API 등 외부 인프라와의 통신을 처리합니다.
함수설명InputOutput
connect_langgraph(builder) 실제 LangGraph나 외부 엔진에 연결하여 그래프 실행 서비스 계층에서 가공된 데이터 외부 엔진 Raw 응답 데이터
handle_external_api(payload) 외부 API와의 통신을 처리 API 페이로드 외부 API 응답 데이터
📄 mock_rag_adapter.py 테스트용 모의 객체
실제 API 호출 없이 테스트용 가짜 답변을 반환합니다. 개발 및 단위 테스트 환경에서 활용합니다.
함수설명InputOutput
get_dummy_response(query) 실제 API 호출 없이 테스트용 가짜 답변 반환 테스트용 쿼리 미리 정의된 더미 답변 데이터

음악 카탈로그 및 계약 데이터 접근을 추상화합니다. Elasticsearch 쿼리, SQLAlchemy ORM, 테이블 JOIN 등 실제 구현 로직을 포함합니다.

📄 base_repository.py 추상 인터페이스
모든 저장소 구현체가 따라야 할 공통 인터페이스를 정의합니다.

search_by_metadata(filters) 필터 조건으로 레코드 조회 필터 조건 객체 조건 부합 레코드 리스트
list_all() 전체 목록 조회 — 전체 레코드 목록

함수설명InputOutput
Elasticsearch 데이터 조회 Elasticsearch에서 데이터를 가져오는 로직 쿼리 파라미터 검색 결과 데이터
ORM 세션 처리 SQLAlchemy / Django ORM을 사용한 실제 DB 세션 처리 DB 연결 정보 DB 세션 객체
Music ⋈ Contract JOIN 조회 Music 테이블과 Contract 테이블을 Join하여 데이터를 가져오는 구체적인 로직 조인 조건 결합된 레코드 결과
📁 RagStateBuilder

상태 조립 계층

LangGraph 기반의 상태 머신을 정의하고 조립합니다. 스키마, 노드, 엣지, 빌더의 4개 파일로 실행 가능한 워크플로우 그래프를 완성합니다.

📄 schema.py 데이터 규격 정의
전체 그래프 흐름에서 유지되고 전달될 데이터의 청사진(TypedDict)을 정의합니다.
클래스/함수설명InputOutput
RagState(TypedDict) 전체 워크플로우에서 공유될 상태 변수들의 집합 질문(String), 채팅 이력(List) 등 사용자 초기 입력 검색된 문서 리스트, 생성된 답변, 재검색 여부 등
📄 nodes.py 구체적인 처리 로직
상태를 입력받아 특정 작업을 수행하고 상태를 업데이트하는 개별 작업 단위입니다.
함수설명InputOutput
retrieve_documents(state) 질문에 관련된 지식을 벡터 DB에서 검색 RagState (현재까지의 상태) RagState (업데이트된 상태)
generate_answer(state) 검색된 컨텍스트와 질문을 바탕으로 LLM을 통해 답변 생성 RagState RagState
grade_documents(state) 검색된 문서들이 질문과 관련 있는지 평가 및 필터링 RagState RagState
transform_query(state) 검색 결과가 부실할 경우 질문을 더 나은 형태로 재작성 RagState RagState
📄 edges.py 경로 및 조건 설정
흐름의 갈림길에서 다음에 어떤 노드로 이동할지 결정하는 조건부 로직을 담당합니다.
함수설명InputOutput
decide_to_generate(state) grade_documents 결과에 따라 답변 생성 또는 질문 재작성 방향 결정 평가 결과가 담긴 RagState 다음 실행할 노드 이름 (String)
check_hallucination(state) 생성된 답변이 문서 내용에 기반하는지 판단하여 최종 완료 여부 결정 생성 결과가 담긴 RagState 다음 실행할 노드 이름 (String)
📄 builder.py 전체 시스템 조립
앞선 구성 요소들을 결합하여 실행 가능한 워크플로우 객체(CompiledGraph)를 생성합니다.
함수설명InputOutput
build_rag_graph() StateGraph를 초기화하고 노드와 엣지를 연결 nodes, edges, schema 구성 요소 StateGraph 객체
compile_graph(graph) 조립된 그래프를 실행 가능한 형태로 컴파일 StateGraph 객체 CompiledGraph (invoke() 실행 가능한 상태 머신)
📁 ContractValidator

검증 로직 계층

계약 데이터의 형식, 비즈니스 로직, RAG 환각 여부를 다층으로 검증하는 파이프라인입니다. 추상 인터페이스를 기반으로 목적별 검증기를 독립적으로 조합합니다.

📄 base_validator.py 추상 인터페이스
모든 검증기들이 공통적으로 가져야 할 규격을 정의합니다.
함수설명InputOutput
validate(data) 입력 데이터의 유효성을 검사하는 기본 추상 메서드 검증 대상 데이터 (계약서 텍스트, 메타데이터 등) 검증 결과 (Boolean) 및 에러 메시지
get_error_report() 검증 실패 시 상세 사유를 반환 — 상세 에러 사유 문자열
📄 format_validator.py 형식 및 스키마 검증
데이터가 시스템이 이해할 수 있는 올바른 형식(JSON, PDF 구조 등)인지 확인합니다.
함수설명InputOutput
check_required_fields(contract_data) 필수 계약 항목(당사자, 일자, 금액 등)의 누락 여부 확인 추출된 계약 데이터 객체 형식 일치 여부 결과
validate_data_types(contract_data) 날짜 형식이나 숫자 단위가 올바른지 확인 계약 데이터 객체 타입 유효성 결과
📄 logic_validator.py 비즈니스 로직 검증
계약 내용이 법적·사업적 정책이나 기존 데이터와 모순되지 않는지 확인합니다.
함수설명InputOutput
verify_contract_period(start_date, end_date) 시작일이 종료일보다 앞서는지 등 기간 논리 확인 시작일, 종료일 기간 논리 타당성 결과
cross_check_with_db(contract_info) 기존 등록 계약과 중복·충돌 조건이 있는지 DB 연동 확인 계약 상세 정보 및 DB 조회 데이터 로직 타당성 결과 및 위반 내역 리포트
check_compliance_rules(text) 필수 문구나 금지 조항이 포함되었는지 검사 계약서 텍스트 컴플라이언스 통과 여부
📄 hallucination_validator.py RAG 전용 환각 검증
RAG 시스템에서 생성된 답변이 원본 계약서 내용에 충실한지(Faithfulness)를 검증합니다.
함수설명InputOutput
verify_grounding(answer, context) 생성된 답변의 모든 문장이 원본 컨텍스트에 근거하는지 확인 생성된 답변(Answer), 참조 문서(Context), 사용자 질문(Query) 환각 점수(Score) 및 통과 여부
check_answer_relevance(answer, query) 질문의 의도에 적합한 답변인지 평가 답변(Answer), 사용자 질문(Query) 관련성 평가 점수

계층 구조
요약

RIMAS RAG 시스템은 5개의 독립적인 계층으로 구성됩니다.
각 계층은 단일 책임 원칙을 따르며, 추상 인터페이스를 통해 유연하게 교체·확장할 수 있습니다.

▶ services RAG 핵심 실행 로직 (인덱싱·임베딩·검색·생성)
▶ adapters 외부 API·라이브러리 의존성 격리 및 교체 가능성 확보
▶ MusicCatalogRepository DB / Elasticsearch 데이터 접근 추상화
▶ RagStateBuilder LangGraph 기반 상태 머신 설계 및 조립
▶ ContractValidator 형식·로직·환각 다층 검증 파이프라인
